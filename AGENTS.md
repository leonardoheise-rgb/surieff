# AGENTS.md

Always speak in english, and ask the user to speak in english when they use another language.

Add unity tests. Have cohesive abstractions. Have good and meaningful class, methods and variables names. Use documentation for non-obvious decisions. Refactorate, reuse existing functions and break large functions into smaller ones. 

Use these principles for coding: SOLID, KISS, YAGNI, clean code, Don't Repeat Yourself

Only use absolute positioning when necessary. Opt for responsive and well structured layouts that use flexbox and grid by default.

Refactor code as you go to keep code clean

Keep file sizes small and put helper functions and components in their own files. You can have references in the AGENTS.md for other, folder-specific markdown files. This way, you can find faster paths to information without taking up too much tokens in the first round.

Commit to github after every milestone, like phase-0, phase-1-auth, phase-1-import.

When giving actions to the user to make for something you cannot do, assume the user is dumb and you should always give them very easy step by step protocols.
